# Markdown 笔记整理最佳实践

本指南用于指导如何更高效地组织和维护你的 Markdown 笔记库，让自动化工具（如本 Skill）更容易理解、分类和利用这些内容。

---

## 一、文件命名规范

良好的文件命名，是后续自动化整理成功与否的关键之一。

### 1.1 推荐命名方式（✅ 建议）

- 使用「日期 + 主题」的形式：
  - `2025-10-21-系统设计原则.md`
  - `2025-11-03-团队周会纪要.md`
- 使用「领域 + 具体主题」的形式：
  - `Python-异步编程实践.md`
  - `产品-用户调研问题清单.md`
- 使用有区分度的关键词：
  - `读书笔记-刻意练习.md`
  - `技术方案-消息队列选型.md`

### 1.2 不推荐命名方式（❌ 避免）

- 过于笼统，无法分类：
  - `笔记.md`
  - `临时记录.md`
  - `杂项.md`
- 含义模糊或多义：
  - `问题汇总.md`（不知是哪个项目、哪类问题）
  - `备忘录.md`
- 仅用数字或无意义字符：
  - `1.md`
  - `aaa.md`

### 1.3 命名小贴士

- 尽量在文件名中包含 1～2 个能代表核心主题的词；
- 如果是长期维护的文档，可以在文件名中加上版本或阶段：
  - `技术路线规划-v1.md`
  - `2025-OKR-复盘-Q1.md`

## 二、目录结构设计

合理的目录结构，可以减少“平铺一大堆文件”的混乱感。

### 2.1 示例结构

```text
notes/
├── 编程技术/
│   ├── Python/
│   │   ├── Python-基础语法.md
│   │   └── Python-异步与并发.md
│   ├── 前端/
│   │   ├── React-组件模式.md
│   │   └── Vue-最佳实践.md
│   └── 数据库/
│       ├── MySQL-索引优化.md
│       └── Redis-缓存策略.md
├── 工作项目/
│   ├── 项目A/
│   │   ├── 需求文档.md
│   │   └── 会议纪要-2025-10-10.md
│   └── 项目B/
